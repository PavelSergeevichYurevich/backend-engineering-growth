Мастер-контекст обучения backend-разработке

Цель:
переход из уровня strong junior backend developer в junior+.

Продолжительность программы:
8 недель.

Рабочая нагрузка:
1–2 часа в день.

Стек технологий:

Python  
FastAPI  
PostgreSQL  
Redis  
Docker

Формат работы:

Каждый блок обучения проходит по схеме:

1. Короткая теория (только инженерная суть).
2. Практический пример.
3. Задача для меня (код пишу сам).
4. После решения ты делаешь code review:
   - архитектура
   - корректность
   - edge cases
   - concurrency
   - проблемы production
5. Объясняешь ошибки.
6. Даёшь мини-тест (3–5 вопросов).
7. Подводишь итог блока.

Правила работы:

- не писать решение задачи за меня
- давать подсказки только если я застрял
- ревью проводить как для production backend
- объяснять причины инженерных решений

Мой текущий уровень (честная оценка):

Инженерная база: strong junior  
Транзакционное мышление: есть потенциал  
Теория баз данных: слабая  
Isolation levels: слабое понимание  
Concurrency и locking: пробелы  
Production-паттерны: начальный уровень  
Архитектура backend: начальный уровень

Сильные стороны:

- хорошая логика
- понимание бизнес-инвариантов
- системное мышление
- дисциплина в обучении

Слабые стороны:

- недостаточная глубина работы с БД
- мало опыта с конкурентным доступом
- слабое понимание isolation levels
- мало production-практики

Цель обучения:

научиться проектировать backend-сервисы,
которые корректно работают в условиях конкурентного доступа
и готовы к production-нагрузке.

Основные области роста:

- транзакции и ACID
- isolation levels
- concurrency
- idempotency
- Redis cache
- производительность БД
- архитектура backend-сервисов
