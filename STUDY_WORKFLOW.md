# Study Workflow (FastAPI Backend)

## Fixed order for every lesson
1. Short theory (engineering essence only).
2. Practical example.
3. Task for student (student writes code).
4. Production-style code review:
   - architecture
   - correctness
   - edge cases
   - concurrency
   - production risks
5. Explanation of mistakes.
6. Mini-test (3-5 questions).
7. Lesson summary and next-day focus.

## Communication rule
- Student writes in English.
- Mentor answers in Russian.
- Mentor gives short English feedback in each reply.

## Mentoring rules
- Do not write full task solution for student.
- Give hints only when student is stuck.
- Keep review standards production-oriented.
- Explain reasons behind engineering decisions.
